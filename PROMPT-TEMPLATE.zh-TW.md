# 互動物件生成提示詞樣板（single 或 2x2）

用途：為 RPG 地圖生成可互動物件（箱子、傳送陣、岩石等）的 single 或 2x2 spritesheet。

建議輸出路徑：
- 你原本想法：`output/object/{物件名稱}/`
- 更穩定命名（推薦）：`output/object/{object-id-kebab-case}/`

範例：
- `output/object/chest-wood/`
- `output/object/warp-circle/`
- `output/object/rock-mossy/`

---

## 通用提示詞（可直接複製）

把下列 `{...}` 參數替換後送給 image generator。

```text
Create exactly one {SHEET_MODE} pixel-art sprite output for a 2.5D side-scrolling RPG interactive object.
SHEET_MODE is either:
- single (1 frame)
- 2x2 (4 frames)

Object identity:
- name: {OBJECT_NAME}
- type: {OBJECT_TYPE}  (chest | warp-circle | rock | sign | etc.)
- biome/style: {BIOME_STYLE}
- material: {MATERIAL}
- color accents: {COLOR_ACCENTS}

Direction policy:
- draw only LEFT-facing and UP-facing native variants when direction matters
- RIGHT-facing must be produced by horizontal mirror of LEFT-facing
- DOWN-facing must be produced by vertical mirror of UP-facing
- keep silhouette mirror-safe (avoid asymmetry that breaks after mirroring)

View and style:
- 2.5D side-view object for a horizontal side-scrolling game
- viewed from a side-facing low-to-mid angle with roughly 45-degree upward tilt
- crisp dark pixel outlines
- consistent 16-bit RPG pixel-art style

Frame plan:
- if SHEET_MODE=single: output exactly one idle frame
- if SHEET_MODE=2x2:
  - frame 1 (top-left): {FRAME_1}
  - frame 2 (top-right): {FRAME_2}
  - frame 3 (bottom-left): {FRAME_3}
  - frame 4 (bottom-right): {FRAME_4}

Consistency rules:
- same object identity across all frames
- same bounding box and same pixel scale in all frames
- entire object must fit fully inside each frame cell
- leave generous magenta margin on all sides
- keep at least 8% to 12% transparent magenta margin from the object silhouette to every cell edge
- no part may touch or cross any cell edge
- no detached particles outside the main silhouette
- avoid semi-transparent magenta fringe on object edges

Background and restrictions:
- background must be 100% solid flat #FF00FF magenta in every cell
- no gradients, no floor plane, no cast shadows
- no text, no labels, no UI, no watermark
- exactly one output matching SHEET_MODE, equal-size cells, no borders between cells
```

## 物件類型速用版

### 1) 箱子（chest）

```text
Create exactly one single pixel-art sprite output for a 2.5D side-scrolling RPG interactive chest.
Wooden chest with iron trims, readable lock plate, adventure-town style.

Direction policy:
- provide native LEFT-facing and UP-facing variants only if needed by engine pipeline
- RIGHT and DOWN are generated by mirror in engine

Frame plan:
- one static closed idle frame only

Keep same chest identity, same scale, same bounding box.
Entire chest fully inside frame with generous magenta margin.
Background is 100% solid #FF00FF magenta.
No text, no UI, no watermark, no edge crossing.
```

### 2) 傳送陣（warp-circle）

```text
Create exactly one 2x2 pixel-art sprite sheet for a 2.5D side-scrolling RPG warp circle object.
Ancient stone magic circle pedestal, cyan-blue runes, shrine atmosphere.

2x2 frame plan:
- top-left: idle faint rune glow
- top-right: brighter rune pulse
- bottom-left: peak energy ring
- bottom-right: settle glow loop frame

Direction policy:
- radial object, no directional variant required

Keep same object identity, same scale, same bounding box in all frames.
Entire object fully inside each cell with generous magenta margin.
Background is 100% solid #FF00FF magenta.
No text, no UI, no watermark, no edge crossing.
```

### 3) 岩石（rock）

```text
Create exactly one single pixel-art sprite output for a 2.5D side-scrolling RPG rock prop.
Mossy field rock, earthy gray-green palette, forest biome.

Direction policy:
- provide native LEFT-facing and UP-facing variants only if needed by engine pipeline
- RIGHT and DOWN are generated by mirror in engine

Frame plan:
- one static idle frame only

Keep same rock identity, same scale, same bounding box.
Entire rock fully inside frame with generous magenta margin.
Background is 100% solid #FF00FF magenta.
No text, no UI, no watermark, no edge crossing.
```

## 建議檔案配置

每個物件一個資料夾：

```text
output/object/{object-id}/
  prompt-used.txt
  pipeline-meta.json
  raw-single-or-2x2.png
  processed/
    object.png
    sheet.png
    gif/
```

備註：
- 箱子、岩石通常可先用 single；若後續要開箱動畫再升級成 2x2。
- 傳送陣通常建議 2x2 以保留動畫感。
- 若物件有朝向差異，建議只畫 LEFT 和 UP，RIGHT/DOWN 由程式鏡像生成。
